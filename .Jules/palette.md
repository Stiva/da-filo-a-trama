## 2024-03-27 - Retain Dynamic ARIA Labels on Toggles
**Learning:** When resolving duplicate `aria-label` properties on a toggle button, retaining the highly descriptive dynamic label (e.g., `aria-label={isEnrolled ? 'Annulla iscrizione' : 'Iscriviti'}`) provides far better context for screen readers than a static label (`aria-label="Iscritto"`), despite standard guidelines typically favoring static labels for `aria-pressed` states.
**Action:** Always prefer dynamic, descriptive `aria-label`s when resolving duplicate properties or enhancing accessibility on toggle buttons, unless there's a specific conflicting issue.
